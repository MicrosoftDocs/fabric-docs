1. Review the detected column settings.
1. If the primary key isn't selected, select the appropriate column as the primary key.
1. Configure columns as required:
   * Mark fields as **Mandatory** if they must contain values.
   * Set fields with distinct values as **Unique Key** to allow unique values.
   * Set a column as an **Identity Column** when you need PowerTable to generate unique, sequential values automatically. Only numeric columns and primary key columns can be configured as identity columns.
1. Modify column properties, including:
   * Length
   * Data type
   * Input type
   * Display name
1. Enter default values for columns, if required.
1. Optionally, select **Add Column** to create additional columns.
1. To create a composite primary key, select **Unique Combination**, and then select two or more columns to define a unique combination key.
